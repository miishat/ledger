<threat_model>
## Security Threat Model (ASVS L1)

| ID | Threat | Severity | Mitigation |
|----|--------|----------|------------|
| T-01 | XSS injection via user-controlled strings (package name, RSU grant names) | MED | React auto-escaping prevents HTML injection; all user strings rendered as text nodes only; no dangerouslySetInnerHTML |
| T-02 | NaN/negative values from invalid numeric inputs breaking calculations | LOW | Validate inputs with parseFloat fallback to 0 on submit; enforce `min="0"` on number inputs; filter NaN in calculation functions |
| T-03 | localStorage corruption causing hydration failure | LOW | Wrap persist hydration in try/catch or rely on Zustand's built-in error boundary; store falls back to default state on parse error |
</threat_model>

  <action>
    Create `src/store/useCompensationStore.ts`.

    Use the exact `create<Type>()(persist(..., { name }))` pattern from useThemeStore.ts.

    Export these TypeScript type/interface definitions:
    - `type VestingPreset = '4yr-1yr-cliff' | '3yr-no-cliff' | 'custom'`
    - `type VestingFrequency = 'monthly' | 'quarterly'`
    - `interface VestingSchedule { preset: VestingPreset; totalVestMonths: number; cliffMonths: number; frequency: VestingFrequency }`
    - `interface RSUGrant { id: string; grantName: string; totalGrantValue: number; vestingSchedule: VestingSchedule; grantStartDate: string }`
    - `interface CompensationPackage { id: string; name: string; baseSalary: number; cashBonusPercent: number; esppContributionPercent: number; esppDiscountPercent: number; rrspMatchPercent: number; rrspMatchCap: number; rsuGrants: RSUGrant[] }`
    - `interface VestEvent { monthIndex: number; label: string; vestValue: number; cumulativeVested: number }`

    Export these standalone pure calculation functions (not inside Zustand state):
    - `calcAnnualBonus(pkg): number` — returns `pkg.baseSalary * (pkg.cashBonusPercent / 100)`
    - `calcAnnualESPP(pkg): number` — returns `pkg.baseSalary * (pkg.esppContributionPercent / 100) * (pkg.esppDiscountPercent / 100)`
    - `calcAnnualRRSP(pkg): number` — returns `Math.min(pkg.baseSalary * (pkg.rrspMatchPercent / 100), pkg.rrspMatchCap)`
    - `calcAnnualRSU(pkg): number` — sums `vestValue` from `generateVestEvents(grant)` events where `monthIndex <= 12`, for all grants in `pkg.rsuGrants`
    - `calcTotalComp(pkg): number` — `pkg.baseSalary + calcAnnualBonus(pkg) + calcAnnualESPP(pkg) + calcAnnualRRSP(pkg) + calcAnnualRSU(pkg)`
    - `generateVestEvents(grant: RSUGrant): VestEvent[]` — vest timeline algorithm (see RESEARCH.md §2):
      - If cliffMonths > 0: add cliff event at monthIndex=cliffMonths, vestValue=totalGrantValue*(cliffMonths/totalVestMonths), label=`Month ${cliffMonths} (Cliff)`
      - freqMonths = frequency==='quarterly' ? 3 : 1
      - Loop from cliffMonths+freqMonths to totalVestMonths step freqMonths
      - Each event vestValue = remainingGrantValue / vestCountPostCliff
      - Track running cumulativeVested across all events

    Zustand state:
    - `primaryPackage: CompensationPackage` — default: id='primary', name='Current Offer', all numeric fields=0, rsuGrants=[]
    - `comparePackage: CompensationPackage | null` — default: null
    - `compareMode: boolean` — default: false

    Zustand actions:
    - `setPrimaryPackage(updates: Partial<CompensationPackage>): void` — merges updates into primaryPackage using spread
    - `setComparePackage(pkg: CompensationPackage | null): void`
    - `toggleCompareMode(): void` — flips compareMode
    - `addRSUGrant(grant: RSUGrant): void` — appends to primaryPackage.rsuGrants
    - `removeRSUGrant(id: string): void` — filters primaryPackage.rsuGrants by id
    - `updateRSUGrant(id: string, updates: Partial<RSUGrant>): void` — maps over rsuGrants, merges updates into matching id

    localStorage name: `'ledger-compensation'`
  </action>
