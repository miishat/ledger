<objective>
Build the RSU/Options equity vesting schedule timeline chart (ComposedChart with cliff marker) and the lightweight compare mode delta view, then integrate both into the Compensation page.
</objective>

<must_haves>
## Truths
- RSU vesting timeline uses Recharts `ComposedChart` with `Bar` for vest events and `Line` for cumulative vested value per D-04.
- The cumulative line stroke is `var(--color-accent)`.
- A `ReferenceLine` marks the cliff month in red (`#ef4444`) with dashed stroke and label "Cliff".
- Vesting preset buttons (D-05) are exactly labeled: "4-Year with 1-Year Cliff", "3-Year, No Cliff", "Custom Schedule".
- Preset buttons have minimum touch target `min-h-[44px]`.
- Compare mode (D-06) button exact label: "Compare Another Offer".
- Delta badges: positive = green (`text-green-600` Geometric / `text-emerald-400` Tactical), negative = red (`text-red-500`), zero = neutral (`text-[var(--color-text-secondary)]`).
- Positive delta label from UI-SPEC: "+$XX,XXX more"; negative: "−$XX,XXX less"; neutral: "Equivalent".
</must_haves>

<threat_model>
## Security Threat Model (ASVS L1)

| ID | Threat | Severity | Mitigation |
|----|--------|----------|------------|
| T-01 | XSS via custom RSU grant names rendered in chart labels or compare view | MED | Render user-supplied grant names using standard React text nodes only; never inject into SVG label prop as raw HTML; Recharts `label` prop with `{ value: grantName }` auto-escapes |
| T-02 | Browser hang via unbounded vest timeline computation (extremely large totalVestMonths or tiny freqMonths) | LOW | Validate: totalVestMonths must be 1–120, frequency must be 'monthly' or 'quarterly'; enforce in CompensationModal input validation before calling addRSUGrant |
</threat_model>
