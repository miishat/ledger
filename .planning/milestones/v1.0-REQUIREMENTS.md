# Requirements

## v1 Core Requirements (Phases 1-6)

### Global Navigation (NAV)

- [x] [NAV-01] Sidebar/Top bar for module switching
- [x] [NAV-02] Dual-theme toggling (Tactical Monospace vs Geometric Abstraction)

### Master Dashboard (DASH)

- [x] [DASH-01] High-level Net Worth aggregation widget
- [x] [DASH-02] Total assets, debts, investment accounts, and bank balances overview widgets
- [x] [DASH-03] High-density Bento Grid layout map accepting child components

### Budgeting Module (BUDGET)

- [x] [BUDGET-01] Monthly cash flow tracking widget
- [x] [BUDGET-02] Income tracking widget
- [x] [BUDGET-03] Categorized expense tracking widget

### Investment Tracker (INVEST)

- [x] [INVEST-01] "Investment Plan vs. Actual" view
- [x] [INVEST-02] Visual comparison of target allocations vs current holdings (using Recharts)

### Future Projections (PROJ)

- [x] [PROJ-01] Visual forecasting tool
- [x] [PROJ-02] Inputs for current savings rate and compounding interest
- [x] [PROJ-03] Graph of estimated future net worth (using Recharts)

### Total Compensation Calculator (COMP)

- [x] [COMP-01] Base salary, cash bonuses, employer-paid benefits breakdown
- [x] [COMP-02] Customizable equity tracker for RSUs/Options vesting schedules
- [x] [COMP-03] Time-based Total Compensation Modes (Current Year vs Next 1 Year), filtering RSU vestings

## v1 Budgeting Automation Requirements (Phases 7-9)

### CSV Import & Triage (CSV)

- [x] [CSV-01] Support pre-defined CSV mappings for RBC, Scotiabank, TD bank
- [x] [CSV-02] Generic column mapper for unknown/unrecognized CSV formats
- [x] [CSV-03] Tinder/Inbox-style review flow for imported uncategorized transactions

### Rules Engine (RULES)

- [x] [RULES-01] Store substring-based categorization rules in Zustand
- [x] [RULES-02] Auto-categorize transactions matching rules on import
- [x] [RULES-03] UI to view, add, and delete categorization rules

### Budgeting Paradigms (PARADIGM)

- [x] [PARADIGM-01] Store selected paradigm (Target-Based, Zero-Based, Ledger Custom) in Zustand
- [x] [PARADIGM-02] Zero-Based mode: strict enforcement that total assigned = income
- [x] [PARADIGM-03] Budget math computes Effective Targets incorporating reallocations
- [x] [PARADIGM-04] Visual reallocation interface for overspending

## v2 Requirements (Deferred)

- [ ] User authentication and cloud sync

## Out of Scope

- [Live Bank Sync (e.g., Plaid)] — Deferred to focus on local-first PWA and core layout foundations.
- [Complex Backend Database] — Starting with Vite SPA + LocalStorage for a fast, client-side PWA architecture.
